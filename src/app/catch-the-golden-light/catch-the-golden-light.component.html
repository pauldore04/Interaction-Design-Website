Documentation

Our first concept was to create a multiplayer game where players would press the button as many times as possible within a specific time frame. The player who achieved more button presses would win. The LED would provide feedback by blinking once if Player 1 won or twice if Player 2 won.

We explored alternative ideas, such as:

  • A timing challenge where users would have to estimate a specific number of seconds accurately. We identified potential issues with providing clear user feedback and explaining the concept.
  • A drinking game where participants would compete to finish a beer faster. Players would press the button at the start and again when finished.

We ultimately decided on creating a game called "Catch the Golden Light." In this game:

  • The LED cycles through various colors
  • Occasionally, with a certain probability, the LED would display gold
  • The user must press the button at the exact moment the gold color appears to win
  • If pressed at the wrong time, the LED flashes red
  • When the player wins, the LED remains gold
  • After winning, the game resets and begins again

We favored this concept because it was a simple game with an intuitive title that required minimal explanation.

After settling on our concept, we created a simple flowchart to guide our programming process.

We initially planned to use mob programming on a large display so everyone could contribute simultaneously. However, we encountered technical difficulties with the display.

As a workaround, all four team members gathered around a single laptop to develop the code. This approach proved challenging as it limited true team collaboration.

After our lunch break, we made some adjustments to the code and modified the colors to improve the game experience.

Following a successful presentation of our project, we had the opportunity to see the creative ideas implemented by the other groups.
